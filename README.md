# 🧠 Disease Prediction System
An **ML-powered, educational web application** that demonstrates how **machine learning, Bayesian probability, and AI** can be used to estimate disease likelihood based on symptoms and diagnostic test results.

This project is designed to make **diagnostic reasoning transparent**, intuitive, and interactive - especially for students, researchers, and developers exploring medical ML concepts.

## 🧮 Bayes' Theorem Explained
Bayes' Theorem updates the probability of a condition after observing new evidence, such as a test result.
### **Formula:**
```java
P(A|B) = [P(B|A) × P(A)] / [P(B|A) × P(A) + P(B|¬A) × P(¬A)]
```
Where:
- **P(A)** → Prior probability
- **P(B|A)** → Sensitivity
- **P(B|¬A)** → False positive rate
- **P(A|B)** → Posterior probability

This project visualizes this shift clearly using charts and explanations.

### 🧮 Bayesian Calculator Datasets

The `hospital_data.csv` used by the Bayesian probability calculator is derived from the following standard public datasets. Each dataset is preprocessed via `data/preprocess.py` to compute real-world `prior_probability`, `sensitivity`, and `specificity` values.

| Disease | Dataset | Source |
|---|---|---|
| Heart Disease | UCI Heart Disease Dataset | https://archive.ics.uci.edu/dataset/45/heart+disease |
| Diabetes | Pima Indians Diabetes Dataset | https://www.kaggle.com/datasets/uciml/pima-indians-diabetes-database |
| Breast Cancer | Breast Cancer Wisconsin (Diagnostic) | https://archive.ics.uci.edu/dataset/17/breast+cancer+wisconsin+diagnostic |

**How values are derived:**
- **`prior_probability`** — Disease prevalence (proportion of positive cases in the dataset)
- **`sensitivity`** — True Positive Rate among confirmed positive cases
- **`specificity`** — True Negative Rate among confirmed negative cases

To regenerate `hospital_data.csv` from the raw datasets:
```bash
python data/preprocess.py
```

> **Note:** Raw dataset files are not committed to this repository due to size and licensing. Download them from the links above and place them in `data/raw/` before running the script.
