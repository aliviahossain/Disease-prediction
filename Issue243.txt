What I changed 
    👇👇👇

I added a new preprocessing module inside backend/preprocessing.

The main file is:
backend/preprocessing/cleaning.py

This file cleans the input data before it goes to the ML prediction model.
It handles disease name, symptoms, age, height, and weight.

I also updated:
backend/preprocessing/__init__.py

This makes the cleaning functions easy to import from other backend files.

I connected this cleaning step with the ML prediction routes in:
backend/routes/ml_routes.py

Now both prediction APIs use the preprocessing code:

/api/ml/predict
/api/ml/predict-multiple

I also added tests in:
backend/tests/test_preprocessing.py

Why I did this:
    👇👇👇

Before this change, the prediction route was taking user input almost directly.
It only had a small check for age.

That can create problems because users may send data in different formats.
For example:

Increased Thirst
increased_thirst
Frequent Urination
frequent-urination

All of these should be cleaned into the format that the ML model understands.

The model expects clean snake_case values like:
increased_thirst
frequent_urination

So I added preprocessing to make the input more stable before prediction.


What the cleaning does:
    👇👇👇

It changes disease and symptom names into snake_case.

Example:
Diabetes Type 2 becomes diabetes_type_2
Increased Thirst becomes increased_thirst

It removes duplicate symptoms.

Example:
["fatigue", "Fatigue"] becomes ["fatigue"]

It removes unknown symptoms that are not part of the model symptom list.

It checks age, height, and weight so invalid values do not go into the model.

Example:
Age cannot be 140.
Height cannot be 10 cm.
Weight cannot be 0 kg.

It also gives clear error messages when the input is not valid.


Why this is useful:
    👇👇👇


This makes the ML prediction pipeline cleaner and safer.

It helps avoid wrong predictions caused by messy input.

It also keeps the route code cleaner because the cleaning logic is now in one place.

If we need to improve input cleaning later, we can update the preprocessing file instead of changing many route files.
